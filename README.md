# Backend Overview

This document describes the `/backend` folder of the Nexus project and explains how each Python code file contributes to the AI mental health companion.

## Purpose

The backend provides:
- a FastAPI service for chat, feedback, and health endpoints
- an emotion prediction pipeline
- response generation and reinforcement learning-based action selection
- dataset preparation and training scripts for model creation

## Folder structure

- `prepare_dataset.py` — prepare and balance raw text/emotion training data
- `train_emotion.py` — train an emotion classification model and vectorizer
- `train_cluster.py` — train a clustering model for conversation behavior grouping
- `requirements.txt` — backend Python dependencies
- `app/` — main application package
  - `main.py` — FastAPI app and router registration
  - `api/` — request/response schemas and route definitions
  - `models/` — reusable model loading and RL agent logic
  - `pipeline/` — NLP, clustering, decision, and response logic
  - `services/` — application orchestration and interaction state management
- `saved_models/` — serialized model artifacts, including `emotion_model.pkl`, `vectorizer.pkl`, and `kmeans.pkl`

## How it works

1. A frontend client sends user messages to the backend via `/chat/`.
2. `handle_chat()` in `app/services/chat_service.py` orchestrates the flow.
3. The input text is analyzed for emotion and confidence using `app/pipeline/nlp_pipeline.py`.
4. Features are built and a cluster is assigned from `app/pipeline/clustering.py`.
5. An action is decided using `app/pipeline/decision_engine.py`.
6. A response text is generated by `app/pipeline/response_builder.py`.
7. The subscription state for the user is saved in `app/services/user_service.py`.
8. Feedback sent to `/feedback/` updates the RL agent `app/models/rl_agent.py`.

## Top-level scripts

### `prepare_dataset.py`

- Reads `data/raw/train.csv`.
- Extracts conversation text and emotion labels.
- Converts labels to emotion strings.
- Balances the dataset across all emotion labels.
- Writes `data/processed/emotion_dataset.csv`.

### `train_emotion.py`

- Loads `data/processed/emotion_dataset.csv`.
- Trains a TF-IDF vectorizer and logistic regression classifier.
- Evaluates performance with `classification_report`.
- Saves `saved_models/emotion_model.pkl` and `saved_models/vectorizer.pkl`.

### `train_cluster.py`

- Loads the processed emotion dataset.
- Builds simple numeric features from emotion, message length, and intensity.
- Trains a KMeans clustering model.
- Saves `saved_models/kmeans.pkl`.

### `requirements.txt`

- Lists the backend dependencies:
  - `fastapi`
  - `uvicorn`
  - `transformers`
  - `torch`

## App package files

### `app/main.py`

- Creates the FastAPI application.
- Enables CORS for all origins.
- Registers three routers:
  - `/chat` from `app.api.routes.chat`
  - `/feedback` from `app.api.routes.feedback`
  - `/health` from `app.api.routes.health`
- Adds a root health message at `/`.

### `app/api/schemas.py`

- Defines request and response Pydantic models:
  - `ChatRequest` for the chat endpoint
  - `ChatResponse` for the expected response format
  - `FeedbackRequest` for feedback submission

### `app/api/routes/chat.py`

- Defines the POST `/chat/` endpoint.
- Forwards the request to `app.services.chat_service.handle_chat()`.

### `app/api/routes/feedback.py`

- Defines the POST `/feedback/` endpoint.
- Forwards feedback to `app.services.feedback_service.handle_feedback()`.

### `app/api/routes/health.py`

- Defines the GET `/health/` endpoint.
- Returns a simple status object.

## Models

### `app/models/emotion_model.py`

- Loads a saved emotion classifier and vectorizer from `saved_models/`.
- Exposes `predict_emotion(text)` that returns:
  - predicted emotion label
  - prediction confidence

### `app/models/rl_agent.py`

- Implements a simple Q-learning agent for selecting response actions.
- Defines action space: `empathy`, `motivation`, `advice`, `distraction`, `clarify`.
- Loads and saves a Q-table at `saved_models/rl/q_table.json`.
- Provides functions for:
  - `choose_action(emotion, cluster, intent)`
  - `update_q(emotion, cluster, intent, action, reward)`

## Pipeline

### `app/pipeline/nlp_pipeline.py`

- Uses `app.models.emotion_model.predict_emotion()` to classify text.
- Normalizes confidence into the range `[0.0, 1.0]`.
- Forces at least `0.55` confidence for non-neutral emotions.
- Assigns a default `intent` of `support`.

### `app/pipeline/feature_builder.py`

- Builds a simple feature vector from the inferred emotion.
- Maps known emotion labels to numeric values.
- Returns a list with a single numeric feature.

### `app/pipeline/clustering.py`

- Loads the saved KMeans model if available.
- Predicts a conversation cluster from feature values.
- Falls back to a deterministic cluster if the model is missing or prediction fails.

### `app/pipeline/decision_engine.py`

- Decides which response action to use.
- Normalizes confidence and applies thresholds:
  - low confidence => `clarify`
  - medium confidence => fallback action based on emotion
  - high confidence => RL agent selected action
- Ensures the chosen action is safe and valid.

### `app/pipeline/response_builder.py`

- Stores a library of response templates organized by action and emotion.
- Generates messages with optional openers, closers, and follow-ups.
- Avoids repeating recent user responses using `LAST_RESPONSES` history.
- Returns a final response string.

## Services

### `app/services/chat_service.py`

- Orchestrates the full chat pipeline:
  1. Validate input.
  2. Analyze text for emotion, confidence, and intent.
  3. Build features and assign a cluster.
  4. Decide an action.
  5. Save interaction state for future feedback.
  6. Generate the conversational response.
- Returns an object containing the full response payload.

### `app/services/feedback_service.py`

- Reads the last stored user interaction.
- Validates the interaction state.
- Clips the reward into `[-1.0, 1.0]`.
- Updates the RL agent with the reward.
- Clears the saved state for that user after learning.

### `app/services/user_service.py`

- Maintains in-memory session state for each user.
- Stores the last interaction metadata:
  - emotion
  - cluster
  - intent
  - action
- Provides helpers to get and clear state.

## Notes and caveats

- The backend currently uses an in-memory `USER_STATE` dictionary, so user feedback state is lost when the server restarts.
- `app/services/chat_service.py` is the core orchestration entrypoint for chat requests.
- Training scripts write model artifacts into `saved_models/` and are intended to be run before the API is used.

## Running the backend

1. Install dependencies from `requirements.txt`.
2. Train or restore required models:
   - run `prepare_dataset.py` to build `data/processed/emotion_dataset.csv`
   - run `train_emotion.py` to generate the emotion classifier and vectorizer
   - run `train_cluster.py` to generate the KMeans cluster model
3. Start the API with:
   - `uvicorn app.main:app --reload`
